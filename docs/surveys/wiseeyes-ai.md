# WiseEyes AI (Beta)

## Overview
WiseEyes AI helps you review element photos and generate a structured quality-check report. It is built for faster field documentation reviews with clear severity and next-step guidance.

## Where to Access WiseEyes
- In a survey: select an element, then choose **Q-Check** in the element panel.
- In the app navigation: open **WiseEyes AI** to view your report library.
- If you do not see WiseEyes AI, your organization may not have it enabled yet.

## Before You Run a Q-Check
1. Open the correct floor plan and select the element.
2. Upload clear photos from multiple angles.
3. Confirm element details are current (status, FoV, notes) for better report context.

## Run a Q-Check
1. In the element panel, click **Q-Check**.
2. Wait while WiseEyes analyzes the selected photos.
3. Review findings in the report preview.
4. Use **Re-analyze** if you add new photos or want a fresh result.
5. Click **View in Reports** to open the full report page.

## What the Report Includes
- Installation overview.
- Detailed findings with severity levels.
- Overall status: **Pass**, **Fail**, or **Conditional Pass**.
- Recommended corrective actions.
- Annotated image previews when available.

## Manage Reports in the WiseEyes Library
- Filter by project, floor plan, and element.
- Search by report title or element name.
- Open a report to rename, review, or archive it.
- Download PDF, share a link, or email the report from the report view.
- Managers can manage organization-wide report share links.

## Common Issues
- **No photos available**: add element photos first in [Link Photos](link-photos.md).
- **Unavailable while offline**: reconnect, then run Q-Check again.
- **Missing WiseEyes access**: ask your Owner or Manager to confirm module access for your account.

## Best Practices
- Use at least three clear photos per element.
- Include one wide-context photo and close-up photos.
- Update element details before running Q-Check.
- Use WiseEyes as guidance and complete your normal review process.
